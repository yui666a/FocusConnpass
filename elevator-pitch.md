# プロダクト概要書：FocusConnpass

## 1. ビジョン
**「"今"の自分に、最適なインスピレーションを。」**
膨大な直近開催イベントの中から、個人の興味と状況に完全にパーソナライズされた「今、参加すべき1件」をAIが瞬時に特定する。

---

## 2. 解決したい課題（ペインポイント）
* **イベント検索のノイズ:** connpassの「直近開催」一覧には、自分の専門外や興味のないイベントが大量に混ざり、探すのが苦痛。
* **参加可否の判断コスト:** 興味があるイベントを見つけても、自分のカレンダー（空き時間）や移動時間を別途確認するのが手間。
* **直前の機会損失:** 予定が空いた直後に、自分にぴったりのイベントを見逃してしまう。

---

## 3. プロダクトコンセプト
**「AIネイティブなイベント・フィルター・エージェント」**
ユーザーが「検索」するのではなく、AIがユーザーの「文脈（Context）」を読み取り、connpass APIから取得した直近データを高速でフィルタリングして提示する。

---

## 4. 主な機能イメージ（AI × MCP）

### ① MCPによる多角的な文脈理解（Context-Awareness）
MCP（Model Context Protocol）を介して、以下のデータをAIの判断材料として統合します。
* **Local Data:** 過去の参加履歴や、最近興味を持っている技術（学習メモ、リポジトリのREADME等）を分析し、推薦スコアを算出。
* **Calendar:** Googleカレンダー等の空き時間を自動参照し、物理的に参加可能なイベントのみを抽出。

### ② AIネイティブな推薦・要約
* **Smart Scoring:** 直近100件のイベント概要をLLMが高速スキャンし、「興味あり/なし」を即座に判定。
* **3行レコメンド:** なぜそのイベントが「今」の自分におすすめなのか、パーソナライズされた理由を3行で解説。
* **物理的参加判定:** 会場場所と現在地を照らし合わせ、「今すぐ出れば間に合います」といったアドバイスを付与。

---

## 5. ターゲット・アーキテクチャ（技術スタック）
* **Frontend:** FastAPI (Web UI / 通知管理)
* **AI Agent:** FastMCP (Python SDK) を活用したMCP Server
* **Data Source:** connpass API (リアルタイム取得)
* **Client:** Claude Desktop / Cursor / 自作Web UI (MCP Host環境)

---

## 6. 将来の拡張性
* **フィードバックループ:** 提案に対する「興味あり/なし」の反応をローカルDBに蓄積し、推薦精度を継続的に向上。
* **アウトプット支援:** 参加したイベントの内容をAIが要約し、自分のメモやブログの下書きを自動生成。
