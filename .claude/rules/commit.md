# コミット規約

## Conventional Commits

すべてのコミットメッセージは以下の形式に従う。

```
<type>: <description>
```

## type 一覧

- `feat`: 新機能
- `fix`: バグ修正
- `refactor`: リファクタリング（機能変更なし）
- `test`: テストの追加・修正
- `docs`: ドキュメントの変更
- `chore`: ビルド・設定・依存関係の変更

## ルール

- description は日本語で簡潔に記述
- 1コミット1目的（複数の変更を混ぜない）
- WIP コミットは避ける
